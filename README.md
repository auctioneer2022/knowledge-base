## 本地开发

### 环境准备

```bash
# 安装依赖（需 Python 3.12+）
pip install mkdocs-material

# 本地预览
mkdocs serve

# 构建静态站点
mkdocs build
```

本地预览默认地址：`http://127.0.0.1:8000`

### 配置说明

主配置文件为 `mkdocs.yml`，关键配置项：

**主题与功能**

- 主题：`material`，中文语言包
- 导航：顶部标签页（sticky）、章节导航、目录展开、返回顶部
- 搜索：中文分词搜索，支持搜索建议和高亮
- 编辑：支持 `content.action.edit` 直接跳转源文件编辑

**Markdown 扩展**

- `admonition` / `pymdownx.details`：提示框与可折叠内容
- `pymdownx.superfences` / `pymdownx.highlight`：代码块与行内高亮
- `pymdownx.tabbed`：标签页切换
- `tables` / `toc`：表格与目录（含永久链接）
- `attr_list` / `def_list`：属性列表与定义列表

**插件**

- `search`：内置中文搜索引擎
- `tags`：标签系统

## 部署

项目通过 GitHub Actions 自动部署至 GitHub Pages：

1. 推送至 `main` 分支自动触发构建
2. CI 环境：`ubuntu-latest` + Python 3.12
3. 构建步骤：安装 `mkdocs-material` → `mkdocs build` → 上传制品 → 部署至 GitHub Pages
4. 也可通过 `workflow_dispatch` 手动触发部署

## 内容维护规范

### 文件格式

- 所有知识内容以 Markdown（`.md`）格式存储于 `docs/` 目录
- 每个文件头部包含 YAML frontmatter，含 `id`、`title`、`category` 字段
- 索引页使用表格列出该模块下所有文件，含编码、名称、发文字号、状态

### 新增内容流程

1. 确定目标模块目录
2. 创建 Markdown 文件，填写 frontmatter 元数据
3. 按照既有文件格式录入法规/准则正文
4. 更新对应模块的 `index.md` 索引页
5. 如需新增导航项，更新 `mkdocs.yml` 中的 `nav` 配置
