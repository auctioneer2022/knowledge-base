# 风险评估方法

> 本文件依据《中国注册会计师审计准则第1211号——通过了解被审计单位及其环境识别和评估重大错报风险》编制，为注册会计师系统开展风险评估工作提供规范指引。

## 一、审计风险模型

### 1.1 基本模型

审计风险是指财务报表存在重大错报时注册会计师发表不恰当审计意见的风险。审计风险模型表述为：

```
审计风险 = 重大错报风险 × 检查风险
```

其中：
- **审计风险**：注册会计师可接受的总体风险水平，通常设定为低水平（如5%）；
- **重大错报风险**：财务报表在审计前存在重大错报的可能性，由被审计单位的情况决定，注册会计师无法控制，只能评估；
- **检查风险**：某一认定存在错报，该错报单独或连同其他错报是重大的，但注册会计师未能发现这种错报的可能性。

### 1.2 模型的应用逻辑

审计风险模型的核心应用逻辑为：

1. **评估重大错报风险**：注册会计师通过了解被审计单位及其环境（包括内部控制），评估财务报表层次和认定层次的重大错报风险；
2. **确定可接受的检查风险**：根据可接受的审计风险水平和评估的重大错报风险，倒推确定可接受的检查风险水平；
3. **设计进一步审计程序**：根据可接受的检查风险水平，设计进一步审计程序的性质、时间安排和范围。

**关键关系**：
- 评估的重大错报风险**越高**，可接受的检查风险**越低**，需要的审计证据**越多**；
- 评估的重大错报风险**越低**，可接受的检查风险**越高**，需要的审计证据**越少**。

## 二、重大错报风险的组成

### 2.1 两个层次

重大错报风险分为两个层次：

**财务报表层次的重大错报风险**：与财务报表整体广泛相关的风险，可能影响多项认定。例如：
- 管理层缺乏诚信；
- 被审计单位面临持续经营困难；
- 经济萧条导致的行业性风险；
- 内部控制环境存在重大缺陷。

**认定层次的重大错报风险**：与各类交易、账户余额和披露的具体认定相关的风险。认定层次的重大错报风险可以进一步细分为固有风险和控制风险。

### 2.2 固有风险

固有风险是指在考虑相关的内部控制之前，某类交易、账户余额或披露的某一认定易于发生错报（该错报单独或连同其他错报可能是重大的）的可能性。

### 2.3 控制风险

控制风险是指某类交易、账户余额或披露的某一认定发生错报（该错报单独或连同其他错报可能是重大的），而该错报没有被被审计单位的内部控制及时防止或发现并纠正的可能性。

## 三、固有风险的识别与评估

### 3.1 行业因素

注册会计师应当了解被审计单位所处行业的状况，包括：

- **行业竞争状况**：竞争激烈的行业可能导致利润压力，增加管理层操纵利润的动机；
- **行业监管环境**：高度监管的行业面临更多的合规风险；
- **行业技术变革**：技术快速迭代的行业面临资产减值和技术淘汰风险；
- **行业周期性**：周期性行业在经济下行时面临更大的经营和财务风险；
- **行业供求关系**：供过于求可能导致存货跌价和收入下降。

### 3.2 经营环境

- **宏观经济环境**：经济下行、利率变动、汇率波动等对财务报表的影响；
- **政策法规变化**：新颁布的法律法规（如税收政策、环保要求）对经营的影响；
- **供应商和客户集中度**：过度依赖单一供应商或客户带来的经营风险；
- **供应链稳定性**：供应链中断风险对存货和成本的影响。

### 3.3 会计估计复杂性

某些交易或事项由于其固有特性，更容易产生错报：

- **高度不确定性的会计估计**：如资产减值准备、预计负债、公允价值计量等；
- **涉及复杂计算的会计处理**：如金融工具估值、养老金负债计算等；
- **需要运用大量判断的领域**：如收入确认时点的判断、资产使用寿命的估计等；
- **关联方交易**：关联方交易的商业理由和定价可能缺乏独立性。

### 3.4 管理层诚信与治理

- **管理层的品格和价值观**：管理层是否存在过度激进的会计处理倾向；
- **管理层面临的压力**：业绩考核压力、融资压力、市场预期压力等；
- **治理结构的有效性**：董事会和审计委员会的独立性和监督能力；
- **管理层的变动**：关键管理人员频繁变动可能暗示存在问题。

## 四、控制风险的评估

### 4.1 评估流程

控制风险的评估遵循以下流程：

```
了解内部控制 → 评估控制设计 → 测试控制运行有效性 → 评估控制风险水平
```

### 4.2 了解内部控制

注册会计师应当了解与审计相关的内部控制，包括：

- **控制环境**：治理结构、组织架构、权责分配、人力资源政策、管理层的理念和经营风格；
- **风险评估过程**：被审计单位如何识别和应对经营风险；
- **信息系统与沟通**：与财务报告相关的信息系统和相关沟通；
- **控制活动**：授权审批、职责分离、实物控制、业绩评价等；
- **对控制的监督**：被审计单位如何监督内部控制的有效性。

### 4.3 评估控制设计

评估控制设计是否合理，即控制如果有效运行，是否能够及时防止或发现并纠正重大错报。

**评估要点**：
- 控制是否与特定认定相关；
- 控制的设计是否能够有效应对识别的风险；
- 控制之间是否存在有效的组合效应。

### 4.4 测试控制运行有效性

当存在下列情形之一时，注册会计师应当测试控制的运行有效性：

- 在评估认定层次重大错报风险时，预期控制的运行是有效的；
- 仅实施实质性程序并不能提供认定层次充分、适当的审计证据。

**测试方法**：
- 询问被审计单位人员；
- 观察特定控制的运用；
- 检查文件和报告；
- 重新执行特定控制。

### 4.5 评估控制风险水平

根据控制测试的结果，评估控制风险水平：

- **控制运行有效**：控制风险可以评估为低水平，注册会计师可以在实质性程序中减少样本量或采用分析性程序；
- **控制运行无效**：控制风险应当评估为高水平，注册会计师应当主要依赖实质性程序获取审计证据；
- **控制设计合理但未有效运行**：需要分析控制失效的原因和影响范围。

## 五、检查风险的确定

### 5.1 确定方法

检查风险是根据审计风险模型倒推确定的：

```
检查风险 = 审计风险 / 重大错报风险
```

在实务中，注册会计师通常将审计风险设定为一个可接受的低水平（如5%），然后根据评估的重大错报风险确定可接受的检查风险。

### 5.2 检查风险与审计程序的关系

| 评估的重大错报风险 | 可接受的检查风险 | 进一步审计程序的性质 | 审计证据的数量 |
|------------------|----------------|-------------------|-------------|
| 高 | 低 | 以细节测试为主，辅以实质性分析程序 | 大量 |
| 中 | 中 | 细节测试与实质性分析程序结合 | 适量 |
| 低 | 高 | 以实质性分析程序为主 | 较少 |

### 5.3 检查风险的控制

注册会计师不能将检查风险降低为零，但可以通过以下方式降低检查风险：

- 设计更有针对性的审计程序；
- 在期末或接近期末实施更多的审计程序；
- 扩大审计程序的范围（增加样本量）；
- 采用更有效的审计方法（如从检查改为重新计算）。

## 六、风险评估程序

### 6.1 询问

询问是风险评估的基本程序，注册会计师应当询问：

- **管理层**：关于财务报表的编制、重大交易的商业理由、舞弊风险等；
- **治理层**：关于对管理层的监督、内部控制环境等；
- **内部审计人员**：关于内部审计发现的问题和内部控制缺陷；
- **其他人员**：如生产人员、销售人员、采购人员等，以获取不同角度的信息。

### 6.2 分析程序

在风险评估阶段运用分析程序，目的是：

- 识别那些可能表明存在重大错报风险的异常变化或关系；
- 帮助了解被审计单位业务的性质和趋势；
- 为后续审计程序的设计提供方向。

### 6.3 观察和检查

- **观察**：观察被审计单位的经营活动和内部控制的运行情况；
- **检查**：检查文件、记录和内部控制手册，阅读管理层和治理层的会议纪要，阅读被审计单位内部生成的报告等；
- **实地察看**：实地察看被审计单位的生产经营场所和办公场所。

## 七、了解被审计单位及其环境

### 7.1 行业状况

- 所处行业的市场供求和竞争状况；
- 行业的生产技术和工艺变化；
- 行业特定的能源消耗和环保要求；
- 行业的整体业绩和趋势。

### 7.2 法律环境

- 适用的财务报告编制基础（如企业会计准则）；
- 影响被审计单位的法律法规（如税法、环保法、劳动法等）；
- 政府政策对经营的影响（如产业政策、贸易政策）；
- 监管机构的监管要求。

### 7.3 目标、战略和相关经营风险

- 被审计单位的经营目标和战略方向；
- 为实现战略目标而面临的经营风险；
- 重大经营变化（如并购重组、业务转型、新业务拓展）带来的风险；
- 持续经营能力的评估。

### 7.4 财务业绩的衡量与评价

- 管理层用于衡量财务业绩的关键指标；
- 管理层和治理层对财务业绩的评价和考核机制；
- 预算、预测和差异分析；
- 分析师报告和信用评级。

### 7.5 内部控制

如前文所述，注册会计师应当从五个要素了解内部控制：控制环境、风险评估过程、信息系统与沟通、控制活动、对控制的监督。

## 八、特别风险的识别与应对

### 8.1 特别风险的定义

特别风险是指注册会计师识别和评估的、根据判断认为需要特别考虑的重大错报风险。

### 8.2 应当考虑为特别风险的事项

注册会计师在判断哪些风险是特别风险时，至少应当考虑下列事项：

- **舞弊风险**：存在舞弊风险的事项天然属于特别风险；
- **与重大非常规交易相关的风险**：如重大关联方交易、企业合并等；
- **与重大会计估计相关的风险**：如高度不确定的公允价值计量、重大或有事项等；
- **与持续经营相关的重大不确定性**；
- **超出正常经营过程的重大关联方交易**。

### 8.3 特别风险的应对

对于特别风险，注册会计师应当：

- 评价相关控制的设计，并确定其是否已得到执行；
- 如果拟信赖针对特别风险的控制，应当测试其运行有效性；
- 如果针对特别风险仅实施实质性程序，应当使用细节测试，或将细节测试与实质性分析程序结合使用；
- 不应依赖以前审计获取的关于特别风险相关控制的审计证据。

## 九、舞弊风险的评估

### 9.1 舞弊三角理论

舞弊的发生通常需要具备以下三个条件（舞弊三角理论）：

#### 动机/压力（Incentive/Pressure）

管理层或其他人员实施舞弊的动机或面临的压力，包括：

- 实现盈利预期或业绩考核目标的压力；
- 满足融资条件或维持上市资格的需要；
- 个人经济利益的驱动（如股权激励、奖金）；
- 经营困难或财务困境导致的压力。

#### 机会（Opportunity）

使舞弊得以实施的环境或条件，包括：

- 内部控制存在重大缺陷；
- 管理层凌驾于内部控制之上；
- 复杂的组织结构或关联方关系使舞弊易于隐藏；
- 缺乏有效的治理层监督。

#### 态度/借口（Attitude/Rationalization）

使管理层或其他人员能够为其舞弊行为进行合理化解释的态度或特征，包括：

- 管理层对财务报告的态度不端正；
- 管理层存在不诚实的历史记录；
- 管理层对审计师或监管机构持对抗态度；
- 管理层认为舞弊行为是"暂时的"或"为了公司好"。

### 9.2 舞弊风险的评估程序

1. **项目组讨论**：项目组成员应当讨论财务报表由于舞弊导致重大错报的可能性；
2. **询问**：询问管理层、治理层、内部审计人员和其他人员关于舞弊风险的看法；
3. **分析程序**：通过异常的数据关系识别可能的舞弊迹象；
4. **其他信息**：考虑在获取其他信息过程中识别的舞弊风险因素。

### 9.3 收入确认的舞弊假定

审计准则要求注册会计师在识别和评估由于舞弊导致重大错报风险时，**应当假定收入确认存在舞弊风险**，并评价哪些收入类型、认定或交易可能产生舞弊风险。如果注册会计师认为收入确认不存在舞弊风险，应当在工作底稿中记录得出该结论的理由。

## 十、风险应对策略

### 10.1 总体应对措施

针对财务报表层次的重大错报风险，注册会计师应当采取以下总体应对措施：

- **向项目组强调保持职业怀疑的必要性**；
- **指派更有经验的审计人员**或利用专家的工作；
- **提供更多的督导**；
- **增加审计程序的不可预见性**（如选择以前未曾测试的项目、调整审计程序的时间等）；
- **对拟实施审计程序的性质、时间安排和范围作出总体修改**。

### 10.2 进一步审计程序

针对认定层次的重大错报风险，注册会计师应当设计和实施进一步审计程序，包括：

#### 控制测试

**目的**：测试内部控制的运行有效性。

**实施条件**：
- 预期控制运行有效；
- 仅实施实质性程序不能提供充分、适当的审计证据。

**方法**：询问、观察、检查、重新执行。

#### 实质性程序

**目的**：发现认定层次的重大错报。

**组成**：
- **细节测试**：对各类交易、账户余额和披露的具体细节进行测试；
- **实质性分析程序**：通过分析数据之间的关系获取审计证据。

**时间安排**：
- 对于高风险领域，应当在期末或接近期末实施实质性程序；
- 对于低风险领域，可以在期中实施实质性程序，并对剩余期间实施补充程序。

## 十一、实务操作要点

### 11.1 关键注意事项

1. **风险评估是持续过程**：风险评估不是仅在审计计划阶段进行的一次性工作，而是贯穿审计全过程的持续活动。当获取新信息时，应当重新评估风险；
2. **职业判断的重要性**：风险评估涉及大量的职业判断，注册会计师应当保持职业怀疑，不应过度依赖以往的审计经验或管理层的声明；
3. **项目组讨论的必要性**：项目组成员之间的充分讨论有助于分享信息、识别风险，特别是关于舞弊风险的讨论；
4. **工作底稿的完整性**：风险评估的过程和结论应当完整记录于审计工作底稿中，包括：了解被审计单位及其环境的关键信息、识别的风险、评估的风险水平、应对措施的设计；
5. **风险与审计证据的关系**：始终牢记审计风险模型的逻辑关系——评估的风险越高，需要的审计证据越多，审计程序越严格；
6. **信息技术环境的影响**：在信息技术环境下，风险评估需要考虑信息系统的一般控制和应用控制，以及自动化处理带来的特殊风险。

### 11.2 常见误区

1. **将风险评估流于形式**：风险评估不应仅是填表和走过场，应当真正理解被审计单位的业务和风险特征；
2. **忽视非财务信息**：风险评估不应仅关注财务数据，非财务信息（如行业趋势、管理层变动、员工士气等）往往是识别风险的重要线索；
3. **过度依赖以往经验**：即使以前年度审计未发现问题，也不意味着本期不存在风险，应当每年重新评估；
4. **忽视小规模企业的特殊性**：小规模企业的内部控制可能不够正式，但不能因此认为不存在控制风险，应当通过其他方式了解控制环境。
