# 众益会计专业知识库 — 产品介绍

---

## 一、产品概述

### 1.1 核心定位

众益会计专业知识库是四川众益会计师事务所自主研发的专业法规与准则知识管理平台，定位为会计师事务所的核心知识基础设施。产品以结构化 Markdown 知识库为底座，通过 MkDocs Material 主题构建现代化 Web 站点，覆盖审计、税务、咨询三大核心业务线，同时作为未来 AI 大模型和智能化审计工具的底层知识供给层。

### 1.2 解决的核心问题

会计师事务所执业过程中面临四大核心知识管理痛点：

**法规分散、检索低效**：执业涉及的法律法规、会计准则、审计准则、税法法规分散于财政部、中注协、人大网等多个权威网站，检索效率低且版本管理困难。本产品将核心法规集中收录至统一平台，通过中文全文搜索与层级化导航，实现一次检索、精准定位。

**准则体系复杂、关联性难以把握**：企业会计准则体系包含基本准则、42 项具体准则、20 项准则解释、40 余项应用指南及多项制度规范，条文间存在大量交叉引用。本产品通过统一的编码体系、完整的索引页和层级化目录组织，清晰呈现准则间的从属与关联关系。

**知识更新滞后**：法规与准则持续修订，传统文档管理方式难以保证时效性。本产品基于 Git 版本控制，配合结构化的更新记录机制，确保知识库的时效性与可追溯性。

**AI 审计工具缺乏标准化知识底座**：未来智能化审计工具需要结构化、机器可读的知识源。本产品从建设之初即采用 Markdown + YAML Front Matter 格式，天然适配 AI 系统的检索增强生成（RAG）、知识图谱构建和推理链训练需求。

### 1.3 核心价值

- **效率提升**：审计人员可在知识库中快速定位适用的准则条文、操作指引和参考模板，将审计计划编制时间缩短 30% 以上
- **质量保障**：标准化的知识组织方式和检查清单为审计质量提供系统性保障
- **知识沉淀**：将散落在个人经验、零散文件中的实务知识体系化、结构化，形成事务所可复用的组织记忆
- **AI 就绪**：知识库的结构和内容组织方式天然适配 AI 系统的知识摄入需求，为后续数字化产品提供高质量的知识供给

## 二、核心技术 / 功能

### 2.1 技术架构

产品采用"内容层—生成层—展示层—部署层"四层技术架构：

| 层级 | 技术组件 | 功能说明 |
|------|----------|----------|
| 内容层 | Markdown + YAML Front Matter | 结构化内容存储，支持元数据管理 |
| 生成层 | MkDocs + Python 3.12 | Markdown 转静态 HTML，支持插件扩展 |
| 展示层 | MkDocs Material 主题 | 响应式 UI、中文搜索、深浅色模式 |
| 部署层 | GitHub Actions + GitHub Pages | 自动化 CI/CD、免费托管、CDN 加速 |

### 2.2 核心功能模块

#### 2.2.1 法律法规模块（19部，100%）

涵盖会计法律（会计法、注册会计师法、企业财务会计报告条例、会计基础工作规范、会计档案管理办法）、公司商事法律（公司法、合伙企业法、企业破产法、外商投资法）、合同与民事法律（民法典合同编/物权编、合同编通则司法解释、担保制度司法解释）、劳动法律（劳动法、劳动合同法、社会保险法）、其他相关法律（反洗钱法、数据安全法、个人信息保护法）。

#### 2.2.2 会计准则模块（92/104项，88.5%）

收录企业会计准则基本准则 1 项、具体准则 CAS 1~42 共 41 项、应用指南 40 余项、准则解释第 1~20 号共 20 项、小企业会计准则 1 项、制度与规范 8 项（含企业内部控制基本规范及配套指引、政府会计准则、民间非营利组织会计制度等）。

#### 2.2.3 审计准则模块（47/50项，94.0%）

收录中国注册会计师审计准则 CSA 1101~5101 系列 37 项、审阅准则 1 项、其他鉴证业务准则 2 项、职业道德守则 5 项、质量管理准则 1 项、监管规则 1 项（鉴证业务基本准则），以及 36 项审计准则应用指南索引。

#### 2.2.4 税法法规模块（7部，100%）

按税种分类收录增值税暂行条例及实施细则、企业所得税法及实施条例、个人所得税法、印花税法、税收征收管理法，以及税收优惠、征管与稽查、国际税收等子模块框架。

#### 2.2.5 行业知识模块（17项，100%）

按国民经济行业分类，覆盖制造业（成本核算、存货审计要点）、房地产（收入确认、土地增值税清算）、高新技术（研发费用归集、高新认定）、批发零售（收入确认、存货管理）、建筑（完工百分比法、工程结算）、餐饮住宿（现金交易、税务风险）、农业（生物资产、政府补助）、行政事业单位（政府会计制度、预算会计）、民间非营利组织（非营利组织会计制度）。

#### 2.2.6 功能特性

- **中文全文搜索**：基于 MkDocs Material 内置搜索插件，支持中文分词、搜索建议、关键词高亮
- **响应式导航**：顶部标签页导航（sticky）、章节侧边栏、目录展开、返回顶部按钮
- **深浅色模式切换**：支持浅色/深色主题自适应
- **Markdown 增强**：支持提示框、可折叠内容、代码块高亮、标签页切换、表格、目录永久链接
- **一键编辑**：每页底部集成"编辑此页"按钮，直达 GitHub 源文件编辑界面
- **自动化部署**：推送至 main 分支即自动触发构建与部署

## 三、创新点

### 3.1 四层递进式知识架构设计

产品采用从底层到顶层的四层递进式知识架构：法规原文层（权威、时效、完整的法律与准则原文）→ 结构化知识层（按主题组织的准则解读与法规速查）→ 场景应用层（可直接执行的实务指引与检查清单）→ AI 应用层（智能工具与推理引擎）。当前已完成法规原文层的大量收录工作，为后续层级的知识提炼与 AI 产品化奠定数据基础。这种架构设计使知识库既能满足当下的法规查阅需求，又能平滑演进至智能化应用阶段。

### 3.2 AI 智能化审计知识底座

知识库定位于 AI 智能化审计工具的底层知识供给层，通过以下设计确保 AI 就绪性：

**结构化元数据**：每条知识文档采用统一的 YAML Front Matter 格式，包含 id、title、category、status、effective_date、related_ids、keywords 等字段，便于 AI 系统解析和索引。

**知识关联网络**：通过元数据中的 related_ids 字段和文档内的交叉引用，构建知识条目之间的关联网络。例如处理"某高新技术企业收入确认"问题时，可沿 CAS-14（收入准则）→ IND-03（高新技术行业知识）→ TAX-02（企业所得税优惠）→ CASE-01（相关审计案例）的路径自动关联知识。

**机器可读格式**：Markdown + YAML Front Matter 的格式天然适配 AI 系统的知识摄入，可直接用于构建知识图谱和检索增强生成（RAG）系统，无需额外的数据转换。

### 3.3 规模化法规收录与质量管控流程

从 CICPA 官方 PDF 等权威渠道批量提取审计准则原文，经过系统性的质量管控流程：

- **编号修复**：修复条文编号断裂（如 CAS01 存货 6 处、CAS22 金融工具 33 处）
- **格式清理**：删除 PDF 抓取残留的独立页码行，恢复行首条文编号后空格，清理中文字符间多余空格
- **版本标注**：每条法规标注发文字号、生效日期、现行有效性状态
- **交叉引用**：索引页建立完整的文件链接和状态标注

### 3.4 "编辑→提交→发布"全自动化流程

基于 Git + GitHub Actions 实现内容编辑到发布的全自动流水线：本地编辑 Markdown → Git 提交 → 推送至 main 分支 → GitHub Actions 自动构建 → 部署至 GitHub Pages。全程无需手动干预，大幅降低内容发布的技术门槛。

## 四、应用场景

### 4.1 执业法规速查

审计、税务、咨询业务人员在项目执行过程中，可快速定位并查阅相关法律法规、会计准则、审计准则及税法法规的完整原文。替代传统纸质查阅或分散的网站搜索，将法规检索时间从数十分钟缩短至数秒。

**典型场景**：某审计项目组在执行制造企业年报审计时，需要确认存货跌价准备的计提依据。审计人员通过知识库搜索"存货跌价准备"，即可直达 CAS 1（存货准则）原文、应用指南及制造业存货审计要点，快速获取完整的审计程序指引。

### 4.2 准则学习与继续教育

事务所内部培训与新员工入职培训的知识参考源，覆盖注册会计师考试核心法规、职业道德守则及质量管理准则。

**典型场景**：新入职审计助理需要系统学习收入确认准则。通过知识库的会计准则模块，可按 CAS 14（收入）→ 应用指南 → 准则解释 → 行业知识（房地产/高新技术收入确认）的路径，建立完整的知识框架。

### 4.3 行业专项审计支持

针对特定行业的审计项目，快速获取该行业的特殊会计处理、审计要点和监管要求。

**典型场景**：事务所首次承接房地产开发企业审计业务。审计经理通过行业知识模块的房地产专题，可快速了解不同销售模式下的收入确认原则、土地增值税清算流程、面积差异处理等关键审计要点，降低因行业经验不足导致的审计风险。

### 4.4 AI 审计工具知识供给

开发中

### 4.5 税务咨询与筹划

税务业务人员通过税法法规模块，快速查询各税种的法律规定、优惠政策和申报要求，为客户提供准确的税务咨询服务。

**典型场景**：客户咨询高新技术企业可享受哪些税收优惠政策。税务顾问通过知识库的高新技术专题和税法法规模块，可快速定位企业所得税研发费用加计扣除政策、高新企业税率优惠、技术转让所得税减免等具体规定。

## 五、解决的行业痛点

### 5.1 法规分散、检索低效

**痛点描述**：会计师事务所执业涉及的法律法规、会计准则、审计准则、税法法规分散于财政部、中注协、人大网等多个权威网站，检索效率低且版本管理困难。传统方式下，审计人员需要在多个网站间反复切换，耗费大量时间定位特定条文。

**解决方案**：本产品将核心法规集中收录至统一平台，通过中文全文搜索与层级化导航，实现一次检索、精准定位。所有法规按"领域—模块—条目"三级分类组织，配合统一编码体系，确保知识条目的唯一标识与快速定位。

### 5.2 准则体系复杂、关联性难以把握

**痛点描述**：企业会计准则体系包含基本准则、42 项具体准则、20 项准则解释、40 余项应用指南及多项制度规范，条文间存在大量交叉引用。审计人员在实际执业中难以快速把握准则间的关联关系和应用逻辑。

**解决方案**：本产品通过统一的编码体系、完整的索引页和层级化目录组织，清晰呈现准则间的从属与关联关系。每个准则条目均标注关联条目编码，便于跨准则查阅和综合应用。

### 5.3 知识更新滞后

**痛点描述**：法规与准则持续修订，传统文档管理方式（如 Word 文档、共享文件夹）难以保证时效性，废止文件和新发布文件容易混淆，存在引用失效法规的风险。

**解决方案**：本产品基于 Git 版本控制，配合结构化的更新记录机制（首页记录每次更新的具体内容），确保知识库的时效性与可追溯性。每条法规均标注现行有效性状态，废止文件明确标注并保留历史版本记录。

### 5.4 事务所知识经验难以沉淀复用

**痛点描述**：资深注册会计师的专业经验往往沉淀在个人笔记或头脑中，难以系统化传承。新员工成长周期长，重复性问题反复咨询，效率低下。

**解决方案**：本产品通过四层知识架构将个人经验转化为组织知识资产。法规原文层确保基础知识的权威性，结构化知识层提炼实务要点，场景应用层（开发中）将知识转化为可直接执行的操作指引，最终实现知识的系统化沉淀和复用。

### 5.5 AI 审计工具缺乏标准化知识底座

**痛点描述**：人工智能在审计领域的应用日益广泛，但 AI 系统需要结构化、高质量、机器可读的知识源。传统 PDF 或网页格式的法规数据无法直接用于 AI 训练和推理。

**解决方案**：本产品从建设之初即采用 Markdown + YAML Front Matter 格式，天然适配 AI 系统的检索增强生成（RAG）、知识图谱构建和推理链训练需求。每条知识条目均包含结构化元数据（编码、分类、关键词、关联条目），可直接作为 AI 产品的知识供给源。

## 六、部署方案

### 6.1 在线部署（当前方案）：GitHub Pages

**适用场景**：公开知识库、无需私有化部署的小型事务所

**技术架构**：

- **托管平台**：GitHub Pages（免费）
- **源码托管**：GitHub 仓库
- **CI/CD**：GitHub Actions
- **域名**：默认 `https://<username>.github.io/<repo>/`，支持自定义域名

**部署要求**：

| 项目 | 要求 |
|------|------|
| GitHub 账号 | 需拥有仓库的写权限 |
| 仓库设置 | 启用 GitHub Pages，源选择"GitHub Actions" |
| 构建环境 | ubuntu-latest + Python 3.12 |

**实施步骤**：

1. Fork 或克隆仓库至个人/组织 GitHub 账号
2. 在仓库 Settings > Pages 中，Source 选择"GitHub Actions"
3. 修改 `mkdocs.yml` 中的 `site_url` 为实际部署地址
4. 推送代码至 `main` 分支，GitHub Actions 自动构建部署
5. （可选）在 Settings > Pages 中配置自定义域名

**资源配置**：无额外服务器资源需求，完全免费。

### 6.2 私有化部署：自有服务器

**适用场景**：对数据安全有严格要求、需要内网访问的大型会计师事务所

**技术要求**：

| 项目 | 最低配置 | 推荐配置 |
|------|----------|----------|
| 服务器 | 1 核 CPU / 1GB 内存 | 2 核 CPU / 4GB 内存 |
| 存储 | 10GB SSD | 50GB SSD |
| 操作系统 | Ubuntu 20.04 LTS | Ubuntu 22.04 LTS |
| Web 服务器 | Nginx / Apache | Nginx |
| Python | 3.12+ | 3.12+ |

**实施步骤**：

1. 在服务器上安装 Python 3.12 和 Git
2. 克隆知识库仓库：`git clone <repo-url>`
3. 安装 MkDocs Material：`pip install mkdocs-material`
4. 构建站点：`mkdocs build`
5. 配置 Nginx 指向 `site/` 目录作为静态文件根目录
6. 设置定时任务（crontab）或 Webhook，定期拉取更新并重新构建

**Nginx 配置示例**：

```nginx
server {
    listen 80;
    server_name knowledge.yourfirm.com;
    root /var/www/knowledge-base/site;
    index index.html;

    location / {
        try_files $uri $uri/ =404;
    }

    # 启用 gzip 压缩
    gzip on;
    gzip_types text/plain text/css application/json application/javascript text/xml;
}
```

### 6.3 容器化部署：Docker

**适用场景**：需要快速部署、环境隔离、便于迁移的场景

**Dockerfile 示例**：

```dockerfile
FROM python:3.12-slim

WORKDIR /app
COPY . /app

RUN pip install --no-cache-dir mkdocs-material

EXPOSE 8000

CMD ["mkdocs", "serve", "--dev-addr", "0.0.0.0:8000"]
```

**构建与运行**：

```bash
# 构建镜像
docker build -t knowledge-base .

# 运行容器（开发模式）
docker run -p 8000:8000 -v $(pwd)/docs:/app/docs knowledge-base

# 构建静态站点
docker run -v $(pwd)/site:/app/site knowledge-base mkdocs build
```

**生产环境部署**：可将构建后的 `site/` 目录挂载至 Nginx 容器或 CDN。

### 6.4 部署方案对比

| 自有服务器 | 中 | 可控 | 中 | 内网部署、大型事务所 |

### 7.4 版本更新策略

**内容版本**：

- 每次重大更新在首页 `index.md` 的"更新记录"中详细记录
- 支持通过 Git 标签（Tag）标记里程碑版本

**产品版本**：

| 版本阶段 | 目标 | 时间规划 |
